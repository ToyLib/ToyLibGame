## 🧸 What is ToyLib?

**ToyLib** is a small, approachable game development library written in **C++**,  
designed for **learning, experimentation, and small-scale games**.

It focuses on:
- clarity over complexity  
- modular design  
- hands-on understanding of game engine fundamentals  

Rather than hiding everything behind heavy frameworks, ToyLib encourages you to  
**see and touch how games are built**.

---

## ✨ Features (planned / in progress)

- Simple and flexible **3D / 2D rendering**
- **Actor / Component** based architecture
- Asset loading (models, textures, sounds)
- Input handling (keyboard, mouse, gamepad)
- Math utilities (vectors, matrices, transforms)
- Camera and scene utilities
- OpenGL + SDL2 based
- Lightweight and easy to integrate into small projects

---

## 🚀 Development Philosophy

ToyLib is developed **together with sample games and prototypes**.

The library and example projects evolve side by side to ensure:
- practical usability
- stable APIs
- real-world testing instead of abstract design

This project is not intended to compete with large engines,  
but to serve as a **friendly playground and foundation** for learning and creation.

---

## 🧩 ToyLib and ToyKit

**ToyLib** is the core, low-level game development library.  
It provides fundamental systems such as rendering, input, math, assets, and core engine architecture.

**ToyKit** is a higher-level gameplay framework built on top of ToyLib.  
It focuses on game flow, scenes, UI helpers, and reusable gameplay patterns.

Their relationship can be summarized as:

- **ToyLib** → *engine foundation*  
  - rendering, input, math, physics, core systems  
  - minimal, modular, and engine-oriented

- **ToyKit** → *gameplay toolkit*  
  - scene management, transitions, UI utilities  
  - higher-level abstractions for making games faster

ToyLib can be used **on its own** for experiments or custom engines,  
while ToyKit is optional and intended to **accelerate game development**.
